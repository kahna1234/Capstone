                           STEP-BY-STEP FIX

Step 1: Open PowerShell / Command Prompt
────────────────────────────────────────────────────────────────────────────

Press: Windows Key + R
Type: powershell
Press: Enter

OR

Click: Start Menu → Search → powershell → Open

═══════════════════════════════════════════════════════════════════════════════

Step 2: Navigate to Frontend Folder
────────────────────────────────────────────────────────────────────────────

Type this command:
  cd D:\Capstone\frontend

Press: Enter

✅ You should be in: D:\Capstone\frontend>

❌ Do NOT do: cd D:\Capstone (missing /frontend)
❌ Do NOT do: cd frontend (from wrong directory)

═══════════════════════════════════════════════════════════════════════════════

Step 3: Verify Files Exist
────────────────────────────────────────────────────────────────────────────

Type: dir

You should see:
  index.html
  css (folder)
  js (folder)
  ... and other files

If NOT, you're in wrong directory. Go back to Step 2.

═══════════════════════════════════════════════════════════════════════════════

Step 4: Start the Server
────────────────────────────────────────────────────────────────────────────

Type this command:
  python -m http.server 3000

Press: Enter

✅ SUCCESS - You should see:
  Serving HTTP on 0.0.0.0 port 3000 (http://0.0.0.0:3000/) ...

❌ ERROR - "Address already in use"?
  Try: python -m http.server 8000

❌ ERROR - "python not found"?
  Python not installed. Install from python.org

═══════════════════════════════════════════════════════════════════════════════

Step 5: Open Web Browser
────────────────────────────────────────────────────────────────────────────

Open: Chrome, Firefox, Edge, or Safari
Type in address bar: http://localhost:3000
Press: Enter

✅ Expected Result:
  ECommerce homepage displays
  Navbar visible
  Products showing
  Styles applied (colors, layout)

❌ Not working?
  - Did server start successfully? (Check Step 4)
  - Are you accessing http://localhost:3000? (Not localhost:3000/frontend)
  - Check browser console (F12) for errors

═══════════════════════════════════════════════════════════════════════════════

Step 6: Verify No 404 Errors
────────────────────────────────────────────────────────────────────────────

Press: F12 (Opens DevTools)
Click: Network tab
Refresh: Ctrl + R

Look for:
  ✅ GREEN rows (200 OK) = Files loaded successfully
  ❌ RED rows (404 Not Found) = Files missing

If all GREEN: You're done! ✅
If any RED: Try clearing cache (Ctrl+Shift+Delete)

                        COMMON MISTAKES TO AVOID

❌ WRONG: python -m http.server 3000 (from D:\Capstone)
✅ CORRECT: cd D:\Capstone\frontend THEN python -m http.server 3000

❌ WRONG: Open browser to http://localhost:3000/frontend
✅ CORRECT: Open browser to http://localhost:3000

❌ WRONG: Use https://localhost:3000
✅ CORRECT: Use http://localhost:3000 (no "s")

❌ WRONG: Type "python" and nothing happens
✅ CORRECT: Check Python is installed: python --version

❌ WRONG: Close terminal after starting server
✅ CORRECT: Keep terminal running while testing
